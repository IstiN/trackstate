# TrackState Repository Scope

- Work inside this repository and its existing structure only.
- Prefer small, composable changes that preserve repository conventions and runtime contracts.
- Treat workflow automation, repository scripts, and developer-facing docs as part of the product surface when the task touches them.
- Do not invent extra services, UI layers, or infrastructure unless the ticket explicitly asks for them.
- When the repository is still sparse, inspect the current files and workflows first, then extend the minimal structure that already exists.
