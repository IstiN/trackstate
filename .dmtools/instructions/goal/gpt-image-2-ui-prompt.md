# GPT Image 2 Prompt: TrackState.AI UI Screens

Use this prompt with GPT Image 2 to create modern UI concept screens for TrackState.AI.

## Prompt

Create a polished product UI concept for **TrackState.AI**, a Git-native Jira-compatible project tracking system. The product is a modern Flutter app that can run as a desktop/web/mobile experience and helps teams manage Epics, Stories, Tasks, Sub-tasks, JQL search, workflows, comments, attachments, Git-backed history, and multi-language project collaboration.

Design multiple distinct screens in one cohesive product style. Include these screens:

1. **Project dashboard** with project health, active epics, status distribution, recent activity, and quick actions.
2. **Issue board** with columns for workflow statuses, rich issue cards, priority indicators, assignees, labels, and drag-and-drop feel.
3. **JQL search and issue list** with a query bar, saved filters, sortable columns, pagination, and Jira-like result density.
4. **Issue detail screen** for a Story with summary, description, acceptance criteria, metadata fields, comments, attachments, linked issues, workflow transition button, and visible parent Epic/Sub-task hierarchy.
5. **Hierarchy tree screen** showing Epic → Story/Task → Sub-task nested structure using Git-folder-inspired visual metaphors.
6. **Project configuration screen** for issue types, statuses, workflows, fields, priorities, versions, and components.
7. **Localization/settings screen** showing language selection, translated project labels, locale-aware date formatting, and language pack status.
8. **Mobile issue detail view** showing the same product adapted to a compact responsive layout.

Visual style:

- modern, premium, fresh, elegant, and visually attractive;
- clean SaaS/productivity app aesthetic with strong information hierarchy;
- subtle depth, soft shadows, rounded cards, refined spacing, and crisp typography;
- use a warm or neutral palette such as charcoal, graphite, off-white, ivory, sand, amber, coral, terracotta, plum, emerald, olive, or mint accents;
- **do not use blue as a primary or accent color**;
- **do not imitate Claude Code or any AI terminal branding/colors**;
- avoid generic corporate dashboard clichés;
- make it feel like a serious professional tool, but still fresh, sexy, and delightful.

Layout direction:

- show desktop web screens as high-fidelity app mockups;
- include one responsive mobile mockup beside the desktop screens;
- use realistic UI text for TrackState.AI, issue keys like `TRACK-1`, `TRACK-2`, `TRACK-3`, and JQL examples like `project = TRACK AND status in ("In Progress", "Review") ORDER BY updated DESC`;
- include subtle multi-language cues such as a language switcher, translated status labels, locale-aware dates, and language pack indicators, but keep the primary mockup text in English for readability;
- show Git-native concepts subtly, for example branch/commit icons, repository selector, and "Synced with Git" status, without making the UI look like a developer-only terminal;
- keep all text legible and aligned;
- make the UI look production-ready, not wireframe.

Composition:

- create a cohesive design board with 6-8 screen mockups arranged on a soft neutral background;
- use realistic spacing and consistent components across screens;
- include a small TrackState.AI wordmark;
- prefer dark-on-light or warm neutral UI, not pure dark mode unless used only for small accents;
- no blue, no neon cyberpunk, no terminal-heavy design, no clutter.

Output style:

- high-resolution product design concept;
- Figma-quality UI mockup;
- crisp vector-like interface details;
- modern app store / landing-page presentation quality.

## Negative prompt

No blue colors, no Claude Code-inspired palette, no terminal-only interface, no cyberpunk, no old Jira clone styling, no cluttered admin panels, no illegible tiny text, no placeholder lorem ipsum, no low-fidelity wireframes, no cartoon mascots, no generic stock dashboard look.
